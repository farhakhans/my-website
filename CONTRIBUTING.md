# Contributing to Physical AI Book

Thank you for your interest in contributing to the Physical AI Book! We welcome contributions from the community to help improve this educational resource.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How to Contribute

### Reporting Issues

- Use the issue tracker to report bugs or suggest features
- Search existing issues before creating a new one
- Provide detailed information about the problem or suggestion
- Include steps to reproduce bugs when possible

### Content Contributions

We welcome contributions to improve the content:

1. **Fixing errors**: Typos, factual errors, broken links
2. **Improving clarity**: Making explanations clearer or more accessible
3. **Adding examples**: Additional code examples or exercises
4. **Expanding content**: New sections or chapters on relevant topics

### Technical Contributions

- Bug fixes in the website code
- Improvements to the build process
- Enhancements to the user interface
- Performance optimizations

## Getting Started

1. Fork the repository
2. Create a new branch for your contribution:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Test your changes locally:
   ```bash
   npm run start
   ```
5. Commit your changes with a clear commit message
6. Push to your fork
7. Create a pull request

## Content Guidelines

### Writing Style

- Use clear, accessible language appropriate for beginner to intermediate learners
- Follow the existing structure and formatting
- Include practical examples with executable code
- Maintain the hands-on learning approach
- Ensure examples reflect real-world Physical AI applications

### Technical Requirements

- All code examples should be executable and produce the described results
- Use Python as the primary language for examples, with JavaScript and C++ for specific use cases
- Include appropriate difficulty indicators for exercises
- Follow the existing formatting and styling conventions

## Pull Request Process

1. Ensure your PR addresses a single issue or adds a single feature
2. Update the README.md if needed
3. Add or update tests if applicable
4. Ensure the build passes
5. Wait for review and address feedback

## Questions?

If you have questions about contributing, feel free to open an issue or contact the maintainers.

Thank you for contributing to the Physical AI Book!
